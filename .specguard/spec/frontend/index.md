## Pre-Development Checklist

- [ ] Read [Directory Structure](./directory-structure.md) before adding or moving frontend files.
- [ ] Read [Component Guidelines](./component-guidelines.md) before changing components.
- [ ] Read [Hook Guidelines](./hook-guidelines.md) before changing hooks or data fetching.
- [ ] Read [State Management](./state-management.md) before changing state flow.
- [ ] Read [Type Safety](./type-safety.md) before changing types, schemas, or validation.
- [ ] Read [Quality Guidelines](./quality-guidelines.md) before implementing frontend behavior.

## Quality Check

- [ ] Re-check changed frontend files against [Quality Guidelines](./quality-guidelines.md).
- [ ] Verify components against [Component Guidelines](./component-guidelines.md).
- [ ] Verify hooks and state flow against [Hook Guidelines](./hook-guidelines.md) and [State Management](./state-management.md).
- [ ] Run the smallest relevant frontend test, type-check, lint, or build command.

---

## How to Fill These Guidelines

For each guideline file:

1. Document your project's **actual conventions** (not ideals)
2. Include **code examples** from your codebase
3. List **forbidden patterns** and why
4. Add **common mistakes** your team has made

The goal is to help AI assistants and new team members understand how YOUR project works.

---

**Language**: All documentation should be written in **English**.
