Persona UI 风格对照板 V2

这版不再只换颜色，而是比较三种不同的产品工作模型：写作台、创作画布、运行控制台。真正的设计方向会决定用户每天面对的是“正文”“灵感关系”还是“任务状态”。

判断标准：不要问哪栏颜色好看，问哪栏更像 Persona 的核心体验。布局、信息密度、主操作和组件形态必须一起决定。

A 会让 Persona 更像写作软件。主页面服务于阅读、编辑、章节组织和长期沉浸；AI 功能作为上下文工具出现。

C 会让 Persona 更像 AI 自动化工具。主页面服务于任务编排、运行观察和错误处理；Workflow Runs 很适合，但整体使用会偏工程化。
